# 双臂机器人控制系统 - 完整文档

## 第一部分：项目概述

### 1. 项目简介

#### 1.1 项目定位与目标

本项目是一个基于 UR7e 和 Robotiq 的双臂机器人控制系统，提供完整的轨迹规划、运动控制和夹爪控制功能。系统采用模块化设计，支持单机部署和分布式部署，适用于研发测试和生产环境。

**核心目标**：
- 提供稳定可靠的双臂机器人控制解决方案
- 支持多种运行模式（模拟硬件、真实硬件、分布式部署）
- 提供易用的API接口（ZMQ、ROS2）
- 集成行为树决策系统，支持复杂任务编排
- 支持MoveIt Servo实时伺服控制，可用于遥操作扩展

#### 1.2 技术栈

- **机器人硬件**：UR7e 机械臂、Robotiq 夹爪
- **操作系统**：Linux
- **机器人框架**：ROS2
- **运动规划**：MoveIt2
- **伺服控制**：MoveIt Servo
- **通信协议**：ZMQ（ZeroMQ）
- **决策系统**：py-trees 行为树框架
- **容器化**：Docker & Docker Compose
- **编程语言**：Python 3、C++

### 2. 系统架构

#### 2.1 整体架构图

**系统分层架构**：

| 层级 | 组件 | 说明 | 通信方式 |
|------|------|------|----------|
| **应用层** | 行为树系统 (UR BT) | 高级任务编排和决策 | ZMQ |
| | 客户端应用 | 用户自定义应用 | ZMQ/ROS2 |
| | 自定义任务 | 特定任务实现 | ROS2 |
| **接口层** | ZMQ 客户端 (轨迹规划) | 轨迹规划请求 | ZMQ REQ/REP |
| | ZMQ 客户端 (夹爪控制) | 夹爪控制请求 | ZMQ REQ/REP |
| | ROS2 节点 | MoveIt Servo等 | ROS2 Topic |
| **规划层**<br/>(规划PC) | UR Move 服务器 | 轨迹规划服务 (端口: 5605) | ZMQ REP |
| | 夹爪ZMQ 服务器 | 夹爪控制服务 (端口: 5630/5640) | ZMQ REP |
| | TF 服务器 | 坐标变换服务 (端口: 5609) | ZMQ REP |
| | MoveIt 规划服务 | 运动规划核心 | ROS2 |
| **驱动层**<br/>(驱动PC) | UR 驱动 | 机器人硬件驱动 | ROS2 |
| | 关节状态发布 | 状态发布 (端口: 5650/5651) | ZMQ PUB |
| | 轨迹执行服务器 | 轨迹执行 (端口: 5660/5661) | ZMQ REP |
| | 夹爪驱动 | 夹爪控制 (端口: 5630/5640) | ZMQ REP |
| **硬件层** | UR7e 机械臂 | 左臂/右臂 | 以太网/串口 |
| | Robotiq 夹爪 | 左夹爪/右夹爪 | 串口 |

**数据流向**：

```
应用层
  ↓ (ZMQ/ROS2)
接口层
  ↓ (ZMQ REQ/REP, ROS2 Topic)
规划层
  ↓ (ZMQ PUB/SUB - 关节状态)
  ↓ (ZMQ REQ/REP - 轨迹执行)
驱动层
  ↓ (以太网/串口)
硬件层
```

**详细架构说明**：

1. **应用层 → 接口层**
   - 行为树系统通过ZMQ客户端发送轨迹规划和夹爪控制请求

2. **接口层 → 规划层**
   - ZMQ客户端连接到规划PC的相应服务端口

3. **规划层 → 驱动层**
   - 关节状态：驱动PC通过ZMQ PUB发布，规划PC通过SUB订阅（端口5650/5651）
   - 轨迹执行：规划PC通过ZMQ REQ请求，驱动PC通过REP响应（端口5660/5661）

4. **驱动层 → 硬件层**
   - UR驱动通过以太网控制UR7e机械臂
   - 夹爪驱动通过UR虚拟的串口控制Robotiq夹爪

#### 2.2 核心模块说明

**1. 驱动层（单臂驱动）**
- **功能**：直接控制UR机器人硬件，管理关节状态和轨迹执行，通过虚拟串口，直接控制末端夹具
- **组件**：
  - 单臂机器人驱动（`single_arm_config`）
- **部署**：每台驱动PC运行一个单臂驱动

**2. 规划层（轨迹规划服务器）**
- **功能**：基于MoveIt进行轨迹规划，协调双臂运动
- **组件**：
  - UR Move服务器
  - MoveIt规划服务
  - TF变换服务器
  - 关节状态桥接（接收驱动PC的状态）
- **部署**：任意一台驱动PC，通过ROS_DOMAIN_ID隔离

**3. 执行层（行为树决策系统）**
- **功能**：高级任务编排和决策
- **组件**：
  - 行为树管理器
  - 行为节点库（运动、夹爪、视觉等）
  - 黑板系统（数据共享）
- **部署**：可在任意PC运行，通过ZMQ连接规划服务器

#### 2.3 数据流与通信架构

**通信方式**：
- **ZMQ**：跨平台通信，支持分布式部署
  - REQ/REP模式：请求-响应服务（轨迹规划、夹爪控制）
  - PUB/SUB模式：数据发布订阅（关节状态）
- **ROS2**：系统内部通信（MoveIt、控制器）

**数据流向**：
1. **关节状态流**：驱动PC → 规划PC（PUB/SUB，端口5650/5651）
2. **轨迹执行流**：规划PC → 驱动PC（REQ/REP，端口5660/5661）
3. **规划请求流**：客户端 → 规划PC（REQ/REP，端口5605）
4. **夹爪控制流**：客户端 → 驱动PC（REQ/REP，端口5630/5640）

## 第三部分：核心功能模块

### 7. UR Move 轨迹规划系统

#### 7.1 功能概述

UR Move是系统的核心轨迹规划模块，提供：
- 基于MoveIt的轨迹规划
- 支持关节空间和笛卡尔空间规划
- 多种规划器（PTP、LIN、OMPL）
- 规划与执行分离
- 实时碰撞检测
- ZMQ和ROS2双接口

#### 7.2 规划器类型

**PTP（Point-to-Point）规划器**：
- **适用场景**：关节空间运动，点到点移动
- **特点**：速度快，路径不保证直线
- **使用场景**：快速移动到目标位置，不关心路径形状

**LIN（Linear）规划器**：
- **适用场景**：笛卡尔空间直线运动
- **特点**：末端执行器沿直线移动
- **使用场景**：精确的直线运动，如抓取、放置

**OMPL规划器**：
- **适用场景**：复杂路径规划，避障
- **特点**：支持多种采样算法，可处理复杂约束
- **使用场景**：复杂环境下的路径规划

#### 7.3 路径点格式

**关节空间路径点（type: "joint"）**：
```json
{
    "waypoint_name": {
        "group": "left_arm",
        "planner": "ptp",
        "type": "joint",
        "joint_names": ["left_joint1", "left_joint2", "left_joint3", 
                       "left_joint4", "left_joint5", "left_joint6"],
        "joint_values": [0.0, -90.0, 90.0, -90.0, 90.0, 0.0],
        "max_velocity_scaling_factor": 0.3,
        "max_acceleration_scaling_factor": 0.3,
        "description": "初始位置"
    }
}
```

**笛卡尔空间路径点（type: "cart"）**：
```json
{
    "waypoint_name": {
        "group": "left_arm",
        "planner": "lin",
        "type": "cart",
        "ik_frame": "left_ee_link",
        "frame_id": "left_base_link",
        "position": [0.6, 0.2, 0.35],
        "orientation": [0.382683, 0.0, 0.0, 0.923880],
        "max_velocity_scaling_factor": 0.1,
        "max_acceleration_scaling_factor": 0.1,
        "description": "抓取位置"
    }
}
```

**字段说明**：
- `group`: 机械臂组名称（"left_arm" 或 "right_arm"）
- `planner`: 规划器类型（"ptp", "lin", "ompl"）
- `type`: 路径点类型（"joint" 或 "cart"）
- `joint_values`: 关节角度值（单位：度）
- `position`: 位置 [x, y, z]（单位：米）
- `orientation`: 姿态四元数 [x, y, z, w]
- `max_velocity_scaling_factor`: 速度缩放因子（0.0-1.0）
- `max_acceleration_scaling_factor`: 加速度缩放因子（0.0-1.0）

#### 7.4 规划与执行流程

**流程说明**：
1. **客户端发送路径点** → UR Move服务器
2. **服务器解析路径点** → 转换为MoveIt格式
3. **MoveIt规划轨迹** → 生成轨迹点序列
4. **碰撞检测** → 验证轨迹安全性
5. **返回规划结果** → 包含轨迹数据或错误信息
6. **执行轨迹** → 发送到驱动PC或本地执行

**规划与执行分离**：
- **规划阶段**：只生成轨迹，不执行
- **执行阶段**：使用规划好的轨迹执行
- **优势**：可以先验证轨迹，再决定是否执行

### 8. MoveIt Servo 伺服控制系统

#### 8.1 功能概述

MoveIt Servo提供实时伺服控制功能，支持：
- **实时遥操作**：通过GUI或外部设备进行实时控制
- **低延迟响应**：高频率命令发布（~250Hz）
- **双模式支持**：Twist模式（笛卡尔空间）和Joint模式（关节空间）
- **安全保护**：碰撞检测、关节限位、奇异性检测
- **平滑控制**：内置信号滤波和平滑算法

**适用场景**：
- 遥操作控制
- 精细调整和定位
- 实时交互式控制
- 教学演示

#### 8.2 控制模式

**Twist模式（笛卡尔空间控制）**：
- **控制方式**：通过Twist命令控制末端执行器在笛卡尔空间的运动
- **命令类型**：`geometry_msgs::msg::TwistStamped`
- **Topic**：`/servo_node/delta_twist_cmds`
- **坐标系**：使用`robot_link_command_frame`（默认使用基座标frame）
- **速度单位**：m/s（线性速度）、rad/s（角速度）

**Joint模式（关节空间控制）**：
- **控制方式**：通过JointJog命令控制单个关节的运动
- **命令类型**：`control_msgs::msg::JointJog`
- **Topic**：`/servo_node/delta_joint_cmds`
- **速度单位**：rad/s

#### 8.3 Servo GUI控制界面

**功能特性**：
- 基于PyQt5的图形化控制界面
- 支持Twist和Joint两种控制模式切换
- 实时按钮控制（按下运动，释放停止）
- 状态显示和模式切换

**启动Servo节点**：
```bash
# 启动单臂驱动（左臂或右臂）
ros2 launch single_arm_config left_arm.launch.py

# 在另一个终端启动Servo节点
ros2 launch single_arm_moveit_config servo.launch.py

# 启动Servo GUI
ros2 run single_arm_moveit_config servo_gui.py
```

**GUI使用说明**：
1. **笛卡尔空间控制**：
   - 使用方向按钮控制末端执行器在X、Y、Z方向的运动
   - 按钮按下时持续运动，释放时停止
   - 速度：0.15 m/s（可在代码中调整）

2. **关节空间控制**：
   - 为每个关节提供+/-按钮
   - 控制单个关节的正向或反向运动
   - 速度：0.01 rad/s（可在代码中调整）

3. **模式切换**：
   - 点击"Switch to Twist Mode"切换到笛卡尔空间控制
   - 点击"Switch to Joint Mode"切换到关节空间控制

4. **安全退出**：
   - 点击"Exit"按钮安全退出
   - 退出前会自动停止所有运动命令

#### 8.4 编程接口

**Python示例（Twist控制）**：
```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import TwistStamped

class ServoController(Node):
    def __init__(self):
        super().__init__('servo_controller')
        self.twist_pub = self.create_publisher(
            TwistStamped,
            '/servo_node/delta_twist_cmds',
            10
        )
    
    def move_forward(self, speed=0.1):
        msg = TwistStamped()
        msg.header.stamp = self.get_clock().now().to_msg()
        msg.header.frame_id = "arm_interface_link"  # robot_link_command_frame
        msg.twist.linear.x = float(speed)
        msg.twist.linear.y = 0.0
        msg.twist.linear.z = 0.0
        self.twist_pub.publish(msg)
```

**Python示例（Joint控制）**：
```python
from control_msgs.msg import JointJog

class ServoController(Node):
    def __init__(self):
        super().__init__('servo_controller')
        self.joint_pub = self.create_publisher(
            JointJog,
            '/servo_node/delta_joint_cmds',
            10
        )
    
    def move_joint(self, joint_index, velocity=0.01):
        msg = JointJog()
        msg.header.stamp = self.get_clock().now().to_msg()
        msg.header.frame_id = "base_link"
        msg.joint_names = [
            "shoulder_pan_joint",
            "shoulder_lift_joint",
            "elbow_joint",
            "wrist_1_joint",
            "wrist_2_joint",
            "wrist_3_joint"
        ]
        msg.velocities = [0.0] * 6
        if 0 <= joint_index < 6:
            msg.velocities[joint_index] = float(velocity)
        self.joint_pub.publish(msg)
```

**模式切换服务**：

from moveit_msgs.srv import ServoCommandType

### 14. 接口选择指南

#### 14.1 使用场景对比

| 场景 | 推荐接口 | 原因 |
|------|---------|------|
| 跨平台开发 | ZMQ接口 | 无需ROS2环境，可在本地运行 |
| ROS2节点开发 | ROS2接口 | 直接集成，性能更好 |
| 行为树系统 | ZMQ接口 | 已在ur_bt中封装 |
| 快速测试 | ROS2命令行 | 方便调试和验证 |
| 分布式部署 | ZMQ接口 | 支持跨网络通信 |
| 实时控制 | ROS2接口 | 延迟更低 |

#### 14.2 性能考虑

**ZMQ接口**：
- 优点：跨平台、易用、支持分布式
- 缺点：需要网络通信，有一定延迟

**ROS2接口**：
- 优点：本地通信，延迟低，性能好
- 缺点：需要ROS2环境，跨平台支持有限

#### 14.3 开发建议

1. **新项目开发**：推荐使用ZMQ接口，更灵活
2. **ROS2节点集成**：使用ROS2接口，性能更好
3. **行为树任务**：使用ZMQ接口，已在框架中封装
4. **快速原型**：使用ZMQ接口，无需复杂环境配置

## 第七部分：故障排查与维护

### 20. 常见问题

#### 20.1 连接问题

**问题**：ZMQ连接失败
- **检查**：服务器地址和端口是否正确
- **检查**：防火墙是否允许端口通信
- **检查**：服务器是否正在运行
- **解决**：使用`telnet`或`nc`测试端口连通性

**问题**：机器人连接失败
- **检查**：机器人IP地址是否正确
- **检查**：网络连接是否正常
- **检查**：机器人是否处于远程控制模式
- **解决**：使用ping测试网络连接

#### 20.2 规划失败问题

**问题**：规划失败，返回错误
- **检查**：路径点格式是否正确
- **检查**：目标位置是否可达
- **检查**：是否有碰撞
- **解决**：调整路径点，使用不同的规划器

**问题**：规划超时
- **检查**：超时时间设置是否合理
- **检查**：目标位置是否过于复杂
- **解决**：增加超时时间，简化路径点

#### 20.3 执行异常问题

**问题**：轨迹执行失败
- **检查**：轨迹是否规划成功
- **检查**：机器人是否在安全状态
- **检查**：是否有急停或错误
- **解决**：检查机器人状态，重新规划

**问题**：执行过程中停止
- **检查**：是否有碰撞
- **检查**：速度/加速度设置是否合理
- **解决**：降低速度，检查路径

#### 20.4 性能优化建议

1. **规划优化**：
   - 使用合适的规划器（PTP用于快速移动，LIN用于精确移动）
   - 合理设置速度/加速度缩放因子
   - 避免过于复杂的路径点序列

2. **网络优化**：
   - 使用有线网络连接
   - 确保网络延迟低
   - 避免网络拥塞
